## 简体中文

**Ping Island 0.0.2 聚焦修复 GitHub Actions 打包产物的可启动性问题。**

### 修复

- 修复通过 GitHub Actions 生成的 unsigned DMG 在部分 macOS 环境下无法启动的问题。
- 调整 unsigned 打包流程，重新签名整个 app bundle，避免主程序与内置 Sparkle framework 在下载后出现签名不兼容。
- 明确 unsigned 发布产物仅用于本地测试，首次打开时 macOS 仍可能要求手动放行或移除 quarantine。

### 说明

- 本次 GitHub Release 仍然不是 notarized 正式分发包。
- 如果 macOS 首次启动时阻止运行，请先尝试右键 **Open**，或前往 **系统设置 > 隐私与安全性** 选择 **“仍要打开”**。

## English

**Ping Island 0.0.2 focuses on fixing launch failures in the unsigned GitHub Actions release artifacts.**

### Fixes

- Fixed a launch failure affecting unsigned DMG builds produced by GitHub Actions on some macOS systems.
- Updated the unsigned packaging flow to re-sign the full app bundle so the main app and embedded Sparkle framework keep a compatible signature after download.
- Clarified that unsigned release artifacts are for local testing only, and macOS may still require a manual allow or quarantine removal on first launch.

### Notes

- This GitHub Release is still not a notarized distribution build.
- If macOS blocks the first launch, try **Open** from Finder's context menu or allow it from **System Settings > Privacy & Security**.
